Key artifacts:
- Kafka topic "logs": JSON messages from app producers.
- Redis (Valkey): short TTL dedupe keys "issue:{service}:{issue}" to avoid repeats.
- OpenAI: returns short JSON { reason, suggested_solution } (stored in DB).
- Postgres tables: incidents (details,json), llm_responses (raw/parsed), incident_events (actions/audit).
- PM2: local process manager used for log source and for restarts/actions.

Flow summary (single line): App/PM2 -> Kafka -> Consumer:detect -> Redis dedupe -> OpenAI -> Execute action (pm2) -> Persist incident & LLM response -> (dashboard/analytics)
